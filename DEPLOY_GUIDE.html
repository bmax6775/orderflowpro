3File Upload Process

üì§ Method 1: cPanel File Manager (Recommended)

Step-by-step process:
  1. Access File Manager
    • Log into cPanel
    • Find "File Manager" under "Files" section
    • Navigate to public_html directory
  2. Create Project Folder (Optional)
    # For subdirectory installation Create new folder: orderdesk/
  3. Upload Source Files
    • Click "Upload" button in File Manager
    • Select all OrderDesk files or upload as ZIP
    • Wait for upload completion
    • Extract ZIP file if uploaded as archive
  4. Set Permissions
    uploads/ folder: 755 or 775 uploads/screenshots/ folder: 755 or 775 config.php: 644 All PHP files: 644

Screenshot: cPanel File Manager with uploaded OrderDesk files

üì§ Method 2: FTP Upload

FTP Settings (from cPanel):
Host: ftp.yourdomain.com or your server IP
Username: your cPanel username
Password: your cPanel password
Port: 21 (or 22 for SFTP)
  1. Download and install FileZilla FTP client
  2. Connect using your hosting FTP credentials
  3. Navigate to /public_html/ on remote server
  4. Upload all OrderDesk files to the directory
  5. Set folder permissions using FTP client

4Database Configuration

üóÑÔ∏è Creating MySQL Database

Note: Database names are usually prefixed with your cPanel username (e.g., username_orderdesk)
  1. Create Database
    • In cPanel, find "MySQL Databases"
    • Enter database name: orderdesk
    • Click "Create Database"
    • Note the full database name (with prefix)
  2. Create Database User
    • In "MySQL Users" section
    • Username: orderdesk_user
    • Generate strong password (save it!)
    • Click "Create User"
  3. Assign User to Database
    • In "Add User to Database" section
    • Select user and database
    • Grant "ALL PRIVILEGES"
    • Click "Make Changes"

Screenshot: cPanel MySQL Databases interface

üì• Importing Database Schema

  1. Access phpMyAdmin
    • Find "phpMyAdmin" in cPanel
    • Select your OrderDesk database
  2. Import database.sql
    • Click "Import" tab
    • Choose database.sql file
    • Set character set to "utf8_general_ci"
    • Click "Go" to import
  3. Verify Import
    • Check that all tables are created
    • Verify demo data is imported
    • Should see: users, orders, pricing_plans, etc.
Success: You should see 11 tables created with demo data including Super Admin account.

5Application Configuration

‚öôÔ∏è Updating config.php

Critical Step: This file contains sensitive database credentials. Keep it secure!
  1. Edit config.php
    • Open config.php in File Manager editor
    • Or download, edit locally, and re-upload
  2. Update Database Settings 
    // Replace these values with your hosting details
$host = 'localhost';                    // Usually 'localhost'
$dbname = 'yourusername_orderdesk';     // Your full database name
$username = 'yourusername_orderdesk_user'; // Your database user
$password = 'your_database_password';   // Your database password
  3. Environment Variables (Optional) 
    // For enhanced security, you can use environment variables
$host = getenv('DB_HOST') ?: 'localhost';
$dbname = getenv('DB_NAME') ?: 'yourusername_orderdesk';
$username = getenv('DB_USER') ?: 'yourusername_orderdesk_user';
$password = getenv('DB_PASS') ?: 'your_database_password';

7SSL Certificate Setup

üîí Why SSL is Important

SSL certificates encrypt data transmission and are essential for:
  • Protecting user login credentials
  • Securing file uploads and data
  • SEO benefits and browser trust
  • Professional appearance

üÜì Free SSL Options

  1. Let's Encrypt (Most Common)
    • Available in most cPanel installations
    • Go to "SSL/TLS" in cPanel
    • Select "Let's Encrypt"
    • Choose your domain and install
  2. Cloudflare Free SSL
    • Sign up for free Cloudflare account
    • Add your domain to Cloudflare
    • Update nameservers
    • Enable SSL in Cloudflare dashboard

üîß Force HTTPS Redirect

Add to your .htaccess file to force HTTPS:

# Force HTTPS redirect
RewriteEngine On
RewriteCond %{HTTPS} off
RewriteRule ^(.*)$ https://%{HTTP_HOST}%{REQUEST_URI} [L,R=301]

9Troubleshooting

üö® Common Issues & Solutions

Symptoms: White page or "Connection failed" message
Solutions:
  • Verify database credentials in config.php
  • Check database name includes username prefix
  • Ensure database user has proper privileges
  • Contact hosting support for database issues

Symptoms: Upload fails or "Permission denied" errors
Solutions:
  • Set uploads/ folder permissions to 755 or 775
  • Check PHP upload limits in hosting settings
  • Verify uploads directory exists and is writable
  • Contact hosting support for permission issues

Symptoms: Server error page instead of OrderDesk
Solutions:
  • Check error logs in cPanel
  • Verify .htaccess file syntax
  • Ensure PHP version compatibility
  • Check file permissions (should be 644 for PHP files)

Symptoms: Analytics pages show blank chart areas
Solutions:
  • Check internet connection for CDN resources
  • Open browser developer tools for JavaScript errors
  • Verify Chart.js library is loading
  • Check if browser blocks external scripts

Security Best Practices

Important Security Steps
  1. Change Default Passwords: Update all demo account passwords immediately
  2. Secure config.php: Ensure proper file permissions (644)
  3. Regular Updates: Keep PHP and hosting environment updated
  4. Backup Security: Store backups securely, not on same server
  5. Monitor Logs: Check access logs for suspicious activity
  6. Strong Passwords: Enforce strong password policy for all users
  7. HTTPS Only: Force SSL for all pages
  8. File Permissions: Regularly audit file and folder permissions
